Coordinate your agent fleet for software delivery.

Refine is not another agent harness — it is the management layer on top of your agents. It turns product feedback into Gaps, dispatches agents through planning and implementation, and keeps files, terminals, nodes, processes, evidence, Git, and review close to your local workflow.

What Refine does

Your team's agentic software delivery system.

01

Empower your team

Extend product feedback and app edits to your whole team. Keep planning, features, Gaps, chat, and human verification in one workflow so work moves without losing context.

02

Agent fleet

Orchestrate your agent fleet across decentralized repositories, branches, processes, and your node cluster. Connect any MCP client to the always-on local endpoint, and cheap feedback loops keep agent work moving while git keeps everyone in sync.

03

Governance

Keep agent work aligned with your product intent, local rules, and requirements. Review stays grounded before changes merge.

Get Started

Install Refine locally, then attach the app you want agents to work on.

Install yourself

raw script 
curl -fsSL /scripts/install.sh | bash

The installer checks the host, installs or repairs missing tools when approved, starts Refine, and opens the browser UI on your machine.

Install with your agent

raw runbook 
Follow instructions found at:
/docs/agent-install.md

Give this URL to Codex, OpenClaw, Claude Code, Gemini, Copilot, or another coding agent so it can install Refine with explicit checks and report exact blockers.

Agent URLs

Direct, non-GitHub URLs for automation.

Fly serves repository files directly, so coding agents can fetch install instructions, scripts, and intent docs without depending on raw GitHub access.

And once Refine is running, the daemon always exposes a Model Context Protocol (MCP) endpoint at /mcp, so any MCP client can read status, Gaps, and Features or drive Refine over the same shared capabilities every other surface uses.
