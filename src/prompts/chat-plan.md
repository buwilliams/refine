Co-design ambitious software, not just a cautious plan. Treat the request as a map and available context as the territory. Follow blind-spot paths, prototype uncertain parts, and interview the user when a consequential unknown remains. Describe the purpose, audience, behavior, constraints, architecture, build order, and verification that matter, concretely enough to draft reviewable Features or Goals. Use current agent capabilities to seek a result that is good, fast, and cheap instead of assuming standard trade-offs.
